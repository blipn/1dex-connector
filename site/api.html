Connecteur gratuit

Documentation parcelles par adresse

Envoyez une adresse française et récupérez les parcelles proches en JSON/GeoJSON, sans clé API. Cette page documente uniquement les routes qui répondent aujourd'hui sur 1dex.fr, avec une console de test, curl, CLI npm et OpenAPI.

Endpoint GET /explore/map-layer/{layer}
Auth Aucune
Host https://1dex.fr
Limite volontaire. Pas de host API séparé, pas de clé privée, pas de route future inventée. Le contrat d'entrée reste volontairement simple: address vers parcelles. Des couches publiques complémentaires et vérifiées existent pour explorer les signaux parcelles_dvf, parcelles_travaux, iris, context et parcelles_labels. Les autres couches Explorer restent hors contrat public lorsqu'elles demandent une session ou un accès produit.

Console

Tester une adresse

Saisissez une adresse. Les paramètres de carte restent optionnels et servent surtout aux intégrations qui ont déjà une vue, un zoom ou un contexte géographique.

GET

/explore/map-layer/parcelles

Retourne les parcelles proches d'une adresse sous forme de FeatureCollection. L'adresse est le seul paramètre métier requis; les autres champs affinent une requête déjà contextualisée par une carte.

ParamètreTypeUsage
layerpath · requisparcelles, parcelles_dvf, parcelles_travaux, iris, context ou parcelles_labels.
addressquery · requisAdresse libre saisie par l'utilisateur.
viewport_render_modequery · optionnelfeatures demande les contours GeoJSON.
city_code, lon, latquery · optionnelContexte géographique déjà connu par votre app.
viewport_bbox, viewport_zoomquery · optionnelFenêtre carte à utiliser pour limiter la réponse.
 
curl "https://1dex.fr/explore/map-layer/parcelles?address=50%20rue%20des%20tanneurs%20aix&viewport_render_mode=features" \
  -H "Accept: application/json"
npm i @1dex-fr/1dex
npx 1dex parcelles "50 rue des tanneurs aix" -f summary
npx 1dex dvf "50 rue des tanneurs aix" -f summary
npx 1dex travaux "50 rue des tanneurs aix" -f summary
npx 1dex parcelles "50 rue des tanneurs aix" --url

Schéma

Réponse

{
  "layerKey": "parcelles",
  "label": "Parcelles",
  "summary": "500 parcelles les plus proches affichées · rayon 1 km",
  "status": "success",
  "warnings": [],
  "data": {
    "type": "FeatureCollection",
    "features": [],
    "selected_radius_meters": 1000,
    "total_parcel_count": 620,
    "visible_parcel_count": 500,
    "render_mode": "features"
  }
}

layerKey identifie la couche demandée, ici parcelles.

status vaut success sur la réponse vérifiée.

data.features contient les parcelles GeoJSON visibles.

warnings reste disponible pour les messages machine-readable.
