Vue d'ensemble

Base URL, authentification et formats.

Le connecteur renvoie du JSON UTF-8 depuis https://1dex.fr. Aucun endpoint privé, clé API ou host API séparé n'est documenté ici.

Quoi

Un connecteur pour requêter la couche parcelles publique visible autour d'une adresse 1dex.

Comment

Une console de test, un contrat HTTP JSON, des SDK JavaScript/Python, une CLI, curl et Go via HTTP standard.

Pourquoi

Permettre aux curieux de tester un appel réel avant de visiter 1dex et d'explorer le produit.

Gratuit

La route parcelles documentée est publique et anonyme au moment de cette publication.

Fonctionnel

La doc se limite au endpoint testé sur 1dex.fr, sans inventer de host API ni de route future.

Progressif

Les autres routes seront ajoutées publiquement seulement quand elles existent et répondent.

Base URL

https://1dex.fr

Base publique utilisée par les SDK, la CLI et la console.

Headers

Accept: application/json

Timeouts

Les connecteurs utilisent un timeout client de 30 secondes par défaut. Les erreurs HTTP sont renvoyées telles que publiées par 1dex.fr.

Endpoints

Route publique fonctionnelle.

Cette documentation ne publie que l'endpoint vérifié sur 1dex.fr. Les autres routes seront ajoutées seulement quand elles seront exposées publiquement.

GET

/adresse/{address_slug}/explore/map-layer/parcelles

Retourne les parcelles visibles autour d'une adresse et d'une vue carte, sous forme de FeatureCollection GeoJSON enrichie.

ParamètrePositionObligatoireDescription
address_slugpathouiSlug d'adresse utilisé par 1dex.
city_codequeryouiCode commune INSEE.
lonqueryouiLongitude du point d'exploration.
latqueryouiLatitude du point d'exploration.
parcel_record_keyquerynonParcelle d'ancrage quand elle est connue.
parcel_phasequerynonPhase de chargement, par exemple initial.
viewport_bboxquerynonBounding box de la carte: minLon,minLat,maxLon,maxLat.
viewport_zoomquerynonNiveau de zoom de la carte.
viewport_render_modequerynonMode de rendu. La valeur vérifiée est features.
 
curl "https://1dex.fr/adresse/10-rue-des-cordeliers-aix-en-provence-13100/explore/map-layer/parcelles?city_code=13001&lon=5.446765371857839&lat=43.52966775616209&parcel_record_key=13001000AS0323&parcel_phase=initial&viewport_bbox=5.44628%2C43.52926%2C5.44725%2C43.53008&viewport_zoom=19.25&viewport_render_mode=features" \
  -H "Accept: application/json"

Réponse: 200.

Réponses

Une réponse map-layer vérifiée.

Le endpoint parcelles renvoie un objet de couche avec statut, résumé, warnings et données GeoJSON.

{
  "layerKey": "parcelles",
  "label": "Parcelles",
  "summary": "54 premières parcelles affichées...",
  "status": "success",
  "warnings": [],
  "data": {
    "type": "FeatureCollection",
    "features": [],
    "selected_radius_meters": 300,
    "total_parcel_count": 54,
    "visible_parcel_count": 54,
    "render_mode": "features"
  }
}

layerKey identifie la couche demandée, ici parcelles.

status vaut success sur la réponse vérifiée.

data.features contient les parcelles GeoJSON visibles dans la vue.

data.progressive_load peut indiquer qu'un complément est disponible.

Erreurs

Codes HTTP et warnings machine-readable.

Les connecteurs lèvent OneDexApiError sur les réponses non-2xx et conservent le statut HTTP, le corps décodé, les headers et le request_id disponible.

CodeUsage
400Paramètre manquant ou invalide.
404Adresse, route ou ressource introuvable.
429Trop de requêtes si une limite publique est appliquée.
5xxErreur temporaire côté service.

Warnings fréquents

La route vérifiée peut renvoyer un tableau warnings. Les connecteurs conservent ce champ sans réinterpréter les messages.
