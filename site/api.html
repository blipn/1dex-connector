API 1dex Connector

L'intégration publique commence par une adresse humaine. L'endpoint documenté ici résout cette adresse côté 1dex et renvoie la couche parcelles en JSON.

Ce qui est testable maintenant. Une requête anonyme vers https://1dex.fr/explore/map-layer/parcelles avec un paramètre address.

Ce qui n'est pas exposé. Pas de host API séparé, pas de clé API, pas de route privée, pas de promesse sur des endpoints futurs.

Base URL https://1dex.fr
Auth Aucune
Format application/json

GET

/explore/map-layer/parcelles

Retourne les parcelles proches d'une adresse sous forme de FeatureCollection. L'adresse est le seul paramètre métier attendu; les autres paramètres servent à optimiser ou cadrer une vue carte.

Paramètres

addressquery · requis

Adresse libre saisie par l'utilisateur, par exemple 50 rue des tanneurs aix.

viewport_render_modequery · optionnel

features demande les contours GeoJSON. Sans valeur, 1dex choisit le rendu adapté.

lon, lat, city_code

Coordonnées et code commune optionnels si votre application les connaît déjà.

viewport_bbox, viewport_zoom

Paramètres de carte optionnels pour limiter la réponse à une fenêtre visible.

curl "https://1dex.fr/explore/map-layer/parcelles?address=50%20rue%20des%20tanneurs%20aix&viewport_render_mode=features" \
  -H "Accept: application/json"

Schéma

Réponse

{
  "layerKey": "parcelles",
  "label": "Parcelles",
  "summary": "500 parcelles les plus proches affichées · rayon 1 km",
  "status": "success",
  "warnings": [],
  "data": {
    "type": "FeatureCollection",
    "features": [],
    "selected_radius_meters": 1000,
    "total_parcel_count": 620,
    "visible_parcel_count": 500,
    "render_mode": "features"
  }
}

layerKey identifie la couche demandée, ici parcelles.

status vaut success sur la réponse vérifiée.

data.features contient les parcelles GeoJSON visibles.

warnings reste disponible pour les messages machine-readable.
