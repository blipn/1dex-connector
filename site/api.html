API 1dex Connector

Une adresse en entrée, une couche parcelles en JSON. La page documente uniquement la route publique qui répond aujourd'hui sur 1dex.fr.

Endpoint GET /explore/map-layer/parcelles
Auth Aucune
Host https://1dex.fr
Limite volontaire. Pas de host API séparé, pas de clé privée, pas de route future inventée. Gratuit ici: résoudre les parcelles autour d'une adresse. Les surfaces privées, payantes ou futures restent hors de cette doc tant qu'elles n'ont pas de contrat public vérifié.

GET

/explore/map-layer/parcelles

Retourne les parcelles proches d'une adresse sous forme de FeatureCollection. L'adresse est le seul paramètre métier requis.

ParamètreTypeUsage
addressquery · requisAdresse libre saisie par l'utilisateur.
viewport_render_modequery · optionnelfeatures demande les contours GeoJSON.
city_code, lon, latquery · optionnelContexte géographique déjà connu par votre app.
viewport_bbox, viewport_zoomquery · optionnelFenêtre carte à utiliser pour limiter la réponse.
 
curl "https://1dex.fr/explore/map-layer/parcelles?address=50%20rue%20des%20tanneurs%20aix&viewport_render_mode=features" \
  -H "Accept: application/json"
npm i @1dex-fr/1dex
npx 1dex parcelles "50 rue des tanneurs aix" -f summary
npx 1dex parcelles "50 rue des tanneurs aix" --url

Schéma

Réponse

{
  "layerKey": "parcelles",
  "label": "Parcelles",
  "summary": "500 parcelles les plus proches affichées · rayon 1 km",
  "status": "success",
  "warnings": [],
  "data": {
    "type": "FeatureCollection",
    "features": [],
    "selected_radius_meters": 1000,
    "total_parcel_count": 620,
    "visible_parcel_count": 500,
    "render_mode": "features"
  }
}

layerKey identifie la couche demandée, ici parcelles.

status vaut success sur la réponse vérifiée.

data.features contient les parcelles GeoJSON visibles.

warnings reste disponible pour les messages machine-readable.
