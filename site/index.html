Quickstart

Une adresse, une réponse exploitable.

Le connecteur garde la même enveloppe de réponse sur les routes principales: `request_id`, `query`, `status`, `source`, `data`, `warnings`, `meta`.

JavaScript

import { OneDexClient } from "@1dex/connector";

const client = new OneDexClient({
  baseUrl: "https://api.1dex.fr",
  apiKey: process.env.ONEDEX_API_KEY,
});

const answer = await client.address.resolve(
  "10 rue de la Paix, Paris"
);

Python

from onedex import OneDexClient
import os

client = OneDexClient(
    base_url="https://api.1dex.fr",
    api_key=os.environ.get("ONEDEX_API_KEY"),
)

answer = client.address.resolve(
    "10 rue de la Paix, Paris"
)

SDK + CLI

Des outils courts, auditables, sans logique cachée.

Chaque package reste un client HTTP: aucune base locale, aucun worker, aucun fichier de données embarqué.

JavaScript / TypeScript

Client `fetch`, types publics, erreurs `OneDexApiError`, support des headers et timeouts.

Python

Client `urllib.request`, API miroir du client JS, compatible scripts et notebooks.

CLI

Commandes pour résoudre, autocompléter, interroger une source et exporter en JSON ou CSV.

Contrat API

Quatre routes pour commencer proprement.

Le contrat OpenAPI public est versionné dans le dépôt et publié avec le site.

GET /v1/address/autocomplete POST /v1/address/resolve POST /v1/address/sources POST /v1/address/sources/{source_key}

Public-safe

Fait pour aider à requêter, pas pour exposer l'interne.

La documentation publique ne publie pas de secrets, de schémas internes, de raw data ni de procédures opérateur.

Authentification. Les démos publiques peuvent être gratuites; les quotas compte passent par `Authorization: Bearer`.

Erreurs. Les warnings restent lisibles par machine pour déboguer sans scraper les messages.

SEO. Le site pointe vers 1dex.fr, l'OpenAPI et le dépôt GitHub.
